# AI Agents Instructions

If you are an AI assistant working on this project (`mini_wiki_2.0`), you **MUST** follow these rules:

**Tech Stack Note:** check the plan for the specified tech stack.

## 1. Single Source of Truth
The file `plan.md` is the single source of truth for the project. Always read it first to understand the current milestone, and read the `journal.md` what has been completed, and what the current state is.

## 2. STRICT Design & UI/UX Rules
- **Aesthetic**: Minimal, clean, terminal themed design. DO NOT USE "shiny or playful" naming conventions in code or UI.
- **Colors**: Bright palettes are not allowed.
- **Visuals/Icons**: NO ICONS NOR EMOJIES.
- **Animations**: ONLY loading animations.
- **Bottom line**: We are building a Terminal User Interface TUI.

## 3. Mandatory Milestone Testing (Browser Recording)
**CRITICAL:** At the completion of EVERY milestone, you MUST TEST!!! NEVER hand off a milestone without testing it thoroughly first.

## 4. Milestone Statuses
The project is explained in the `plan.md` file. Each milestone must have one of these statuses in the `journal.md`:
- `PLANNING`
- `INPROGRESS`
- `NOT_STARTED`
- `HANDOFF`
- `VERIFIED_BY_USER` (This means the milestone is entirely complete and approved).
**ASK ALL THE QUESTIONS YOU NEED TO ASK ON THE SPOT!!!!**

## 5. Iteration History Logging
A `journal.md` file must be created serves as a history of iterations. 
Every time you make an iteration (write code, debug, set up infrastructure, etc.), you **MUST** manually edit `journal.md` to log what you did under the current milestone.

**Your log entry must include:**
- What you did/accomplished.
- What you struggled with or broke (what you "fucked up"), so the next agent has full context and doesn't repeat your mistakes.

Never finish your task without updating the iteration history in `journal.md`. Keep your logs clear and concise.
